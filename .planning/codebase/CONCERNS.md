# Technical Concerns

**Analysis Date:** 2026-05-03

## Summary

This is a very early-stage scaffold. Almost no product code exists — the codebase is the `create-next-app` default template with Convex and Clerk dependencies installed but not wired up. The MVP.md describes a substantial feature set that is entirely unbuilt. The concerns below reflect what needs to be addressed before any real development proceeds.

---

## Critical (Blockers)

### 1. Auth Not Configured
- **File:** `convex/auth.config.ts`
- **Issue:** The Clerk JWT provider is commented out. Convex has no auth provider. All `ctx.auth.getUserIdentity()` calls will return `null` in production.
- **Impact:** Any feature requiring user identity (likes, unlocks, bookmarks, dashboard) will silently fail or return `null` user.
- **Fix:** Uncomment and configure the Clerk provider once a Clerk app is created and `CLERK_JWT_ISSUER_DOMAIN` env var is set.

### 2. No Providers in Root Layout
- **File:** `src/app/layout.tsx`
- **Issue:** No `ConvexProvider` or `ClerkProvider` wraps the app. The frontend cannot connect to the Convex backend or Clerk auth at all.
- **Impact:** All Convex hooks (`useQuery`, `useMutation`) and Clerk hooks (`useUser`, `useAuth`) will throw at runtime.
- **Fix:** Add `<ConvexProvider client={convex}>` and `<ClerkProvider>` to `layout.tsx`.

### 3. Placeholder Schema and Functions
- **Files:** `convex/schema.ts`, `convex/myFunctions.ts`
- **Issue:** The only Convex table is `numbers` and the only functions are the starter demo functions (listNumbers, addNumber, myAction). None of the MVP tables (prompts, users, unlocks, bookmarks) exist.
- **Impact:** Zero backend functionality is implemented.
- **Fix:** Replace with domain schema (prompts, categories, unlocks, bookmarks tables) and implement real functions.

---

## High Priority

### 4. Dual Lockfiles
- **Files:** `pnpm-lock.yaml`, `package-lock.json`
- **Issue:** Both pnpm and npm lockfiles are present. This indicates `npm install` was run at some point despite pnpm being the primary package manager (as declared in STACK.md). Dependency resolution may diverge between environments.
- **Impact:** CI/CD or another developer running `npm install` vs `pnpm install` may get different dependency versions.
- **Fix:** Delete `package-lock.json` and add it to `.gitignore`. Enforce pnpm-only via `engines` in `package.json`.

### 5. Missing Dependencies (per MVP.md)
- **Issue:** The following are listed in MVP.md but not installed:
  - `@upstash/redis` / `@upstash/ratelimit` — rate limiting
  - `posthog-js` or `posthog-node` — analytics
  - React Query (`@tanstack/react-query`) — server state management
  - No Gumroad webhook handling library (likely raw `fetch`, but no validation utilities)
- **Impact:** These integrations can't be built until dependencies are added.

### 6. `src/app/page.tsx` Is Default Template
- **Issue:** The home page is the verbatim `create-next-app` template — it links to Next.js documentation and has Vercel branding. No product content exists.
- **Impact:** The app has no functional UI whatsoever.

### 7. Default App Metadata
- **File:** `src/app/layout.tsx`
- **Issue:** `title: "Create Next App"` and `description: "Generated by create next app"` — the generic CNA metadata has not been updated.
- **Impact:** Bad SEO and confusing tab title for end users.

---

## Medium Priority

### 8. No Middleware
- **Issue:** No `src/middleware.ts` exists. The MVP requires Clerk auth protection on `/dashboard/*` and `/api/*` routes.
- **Impact:** Dashboard and API routes are publicly accessible.
- **Fix:** Create `src/middleware.ts` with Clerk's `clerkMiddleware` protecting the appropriate routes.

### 9. No Error Handling
- **Issue:** No `error.tsx`, no `not-found.tsx`, no global error boundaries. Default Next.js error pages will be shown.

### 10. No Image Optimization Setup for Convex Storage
- **Issue:** The MVP calls for serving images from Convex File Storage via their global edge CDN. No image domain is configured in `next.config.ts` for `next/image` to accept Convex storage URLs.
- **Fix:** Add the Convex storage domain to `images.remotePatterns` in `next.config.ts`.

### 11. No SEO Infrastructure
- **Issue:** MVP requires ISR pages, dynamic sitemap, `robots.txt`, and per-prompt OG images. None of these exist yet.
- **Files needed:** `src/app/sitemap.ts`, `src/app/robots.ts`, `src/app/prompts/[category]/[subcategory]/[slug]/opengraph-image.tsx`

---

## Low Priority / Future

### 12. No Tests
- **Issue:** Zero test files. No testing framework configured (Jest, Vitest, Playwright). See TESTING.md for details.
- **Impact:** No safety net for regressions.

### 13. No CI/CD
- **Issue:** No GitHub Actions workflows or Netlify config files present.

### 14. Public Static Assets Are Defaults
- **Files:** `public/*.svg` — all default Next.js/Vercel SVG assets, not product branding.

### 15. `convex/myFunctions.ts` Name
- **Issue:** The starter file is named `myFunctions.ts` — a placeholder name that should be replaced with domain-specific files (`prompts.ts`, `users.ts`, etc.) once real Convex functions are written.

---

## Security Notes

- No secrets are hardcoded in source files (`.env` and `.env.local` contain secrets but are gitignored).
- No API routes exist yet — when `/api/webhook/route.ts` is created for Gumroad Ping, it must be protected with Upstash rate limiting and Gumroad signature verification before going to production.
- Clerk route protection (middleware) must be in place before any protected pages are deployed.
- Image watermarking is handled manually pre-upload (outside this codebase) — no code risk.
